  <action>
Create `content-creation/bin/content-tools.cjs` as a CJS Node.js script (~300 lines).

Architecture (mirrors gsd-tools.cjs):
- `#!/usr/bin/env node` + `'use strict'`
- Constants: `CONTENT_DIR = '.content'`, `GLOBAL_LAYER = path.join(os.homedir(), '.claude', 'content-creation')`
- `output(result)` helper: JSON.stringify to stdout; if >50KB write to tmp file and output `@file:/path`
- `fail(msg)` helper: `process.stderr.write(msg)` + `process.exit(1)`
- `--cwd <path>` flag parsed first from argv; override `process.cwd()` with this value for all path operations
- CLI router: `switch (args[0])` dispatching to named functions

Subcommands to implement:
1. **`init`**: Check `fs.existsSync(path.join(GLOBAL_LAYER, 'guidance.md'))` for global layer. Check `fs.existsSync(path.join(cwd, CONTENT_DIR, 'state.json'))` for active project. If active, read state.json and return slug + current.step. Return `{ global_layer_exists, active_project_exists, active_project_slug?, active_project_step? }`.

2. **`generate-slug "<text>"`**: Implement slug algorithm: `.toLowerCase().replace(/[^a-z0-9]+/g, '-').replace(/^-+|-+$/g, '')`. Then truncate at 60 chars: find last `-` at or before index 60, truncate there. Return `{ slug }`.

3. **`project-init "<subject>"`**: Call generate-slug on subject. If `.content/state.json` exists, call project-archive first. Create dirs: `.content/`, `.content/references/`, `.content/inspiration/`, `.content/archive/`, `{slug}/`. Write initial state.json using the exact schema from CONTEXT.md with subject populated. Return `{ project_dir: ".content", output_dir: "{slug}", slug }`.

4. **`state-read`**: Read `.content/state.json`, return parsed object. If missing: `{ error: "no active project" }`.

5. **`state-checkpoint`**: Parse `--step`, `--sub-step`, `--field`, `--value` from args. Read current state.json, update `state.current = { step, sub_step }`, update `state.steps[step].inputs[field] = JSON.parse(value)` (if field provided), update `state.steps[step].timestamp = new Date().toISOString()`. Write atomically with `fs.writeFileSync`. Return `{ ok: true }`.

6. **`project-archive`**: Read state.json to get current slug. Create `.content/archive/{slug}/`. Copy everything in `.content/` (excluding `archive/` itself) into `.content/archive/{slug}/` using `fs.cpSync`. Wipe `.content/references/`, `.content/inspiration/`, delete `.content/state.json`. Return `{ archived_to: ".content/archive/{slug}" }`.

7. **`add-reference`**: Parse `--type` (reference|inspiration), `--source` (URL or empty), `--content` (text), `--step` (defaults to current step from state). Count existing files in `.content/{type}s/` to determine next index (padded 3 digits). Write `.content/{type}s/{index}.md` with frontmatter (type, index, source_url, added_at, step) + content. Read state.json, append `"{type}s/{index}.md"` to `state.steps[step].inputs.{type}s` array. Write state.json atomically. Return `{ file: "{type}s/{index}.md", index }`.

Error handling: All subcommands catch errors and call `fail(err.message)`.

2. Add an `installBinFile(file)` function that copies `path.join(SRC, file)` to `path.join(TARGET, file)` unconditionally (bin files are always overwritten on re-run — they are code, not creator content):
```javascript
function installBinFile(file) {
  const src  = path.join(SRC, file);
  const dest = path.join(TARGET, file);
  if (!fs.existsSync(src)) {
    console.log(`  ! Skipping ${file} (not yet in source)`);
    return;

  fs.copyFileSync(src, dest);
  console.log(`  ✓ ${file} → ${dest} (bin)`);
}
```
