  <action>
Create `content-creation/workflows/content-resume.md`. This file is installed to `~/.claude/content-creation/workflows/content-resume.md` by `install-content.js`.

The workflow has these steps:

---

**Step 0 — Read state**

```bash
PROJECT_ROOT=$(pwd)
STATE=$(node "$HOME/.claude/content-creation/bin/content-tools.cjs" --cwd "$PROJECT_ROOT" state-read)
```

Parse the JSON result:
- If `error` field is present (no active project): Display:
  ```
  No active content project found in this directory.
  Run /content:new to start a project.
  ```
  Stop here.

- If state.json exists but is corrupted (JSON parse fails): Display:
  ```
  state.json could not be parsed. The file may be corrupted.
  Last known state: .content/state.json

  Options:
  1. Inspect .content/state.json directly and fix the JSON
  2. Run /content:new to start a fresh project (existing state will be archived)
  ```
  Stop here.

**Step 1 — Display full context refresh**

Read the state and display a structured summary. For the `subject_capture` step inputs, expand references and inspiration by reading each file:

For each path in `state.steps.subject_capture.inputs.references`:
- Read `.content/{path}` using the Read tool
- Extract: source_url from frontmatter, and full body content

For each path in `state.steps.subject_capture.inputs.inspiration`:
- Read `.content/{path}` using the Read tool
- Extract: source_url from frontmatter, and first 200 chars of body

Display:
```
--- Project Context ---

Subject: {state.subject}
Slug: {state.slug}
Created: {state.created_at}

Current step: {state.current.step} / {state.current.sub_step}

--- Captured Inputs ---

Subject: {state.steps.subject_capture.inputs.subject}

Intent: {state.steps.subject_capture.inputs.intent || "(not yet captured)"}

Knowledge: {state.steps.subject_capture.inputs.knowledge || "(not yet captured)"}

References ({count}):
{For each reference file: display source_url + first 300 chars of content}

Inspiration ({count}):
{For each inspiration file: display source_url + first 200 chars of content}

--- Pipeline Status ---

{For each step in ["subject_capture", "alignment", "knowledge", "research", "ideation", "draft", "curation", "linkedin"]:
  ✓ {step} (if completed: true)
  → {step} (if current step)
  ○ {step} (if not yet started)
}
```

**Step 2 — Continue from current step**

Based on `state.current.step`:

- `subject_capture` (not completed): The first session is incomplete. Continue the `/content:new` session from sub_step `{state.current.sub_step}`. Pick up the sequence from where it stopped (e.g., if sub_step is `intent_captured`, next question is knowledge).
  - Re-run the appropriate remaining questions from the content-new workflow. Pass `--cwd "$PROJECT_ROOT"` to all content-tools.cjs calls.

- `subject_capture` (completed, next is `alignment`): Display:
  ```
  Subject capture is complete. The next step is alignment — checking your subject against editorial guidance.
  The alignment step will be available in Phase 3.
  ```

- Any other completed step: Display the current step name and note that the corresponding phase handles it:
  ```
  Current step: {step_name}
  This step is handled by Phase {N}. Run the appropriate content command for this step.
  ```

---

Important: When resuming the subject_capture step mid-session, do NOT re-ask questions that are already captured (sub_step checkpoint tells you which fields are populated). Read state.json inputs to know what was already answered and resume from the first unanswered field.
  </action>
